## 1. Исправление скрипта start-all

### Проблема
`pnpm start:all` открывал 3 окна PowerShell, но сервисы не запускались — `ERR_CONNECTION_REFUSED` на localhost.

### Причины
1. **uvicorn** — в новом окне PowerShell `uvicorn` мог не быть в PATH. На Windows при `pip install` исполняемый файл попадает в `Scripts\`, который не всегда в PATH дочернего процесса.
2. **Рабочая директория** — `Start-Process` не задавал `-WorkingDirectory`, дочерний процесс мог стартовать в неправильной папке.
3. **Путь к harvesters** — `$root\services\harvesters` мог некорректно раскрываться в строке `-Command`.

### Исправления

**scripts/start-all.ps1:**
- Добавлен `-WorkingDirectory` для каждого `Start-Process`
- Замена `uvicorn` на `python -m uvicorn` (надёжнее на Windows)
- Использование `Join-Path` для пути к harvesters
- `Set-Location` вместо `cd` для совместимости
- Подсказка при ошибке: команды для ручного запуска

**scripts/start-all.bat:**
- Замена `uvicorn` на `python -m uvicorn`

### Ручной запуск (если скрипт не сработал)
В трёх отдельных терминалах:
```powershell
# 1. AI Engine
pnpm --filter @omega/core dev

# 2. Web
pnpm --filter @omega/web dev

# 3. Harvesters
cd services\harvesters
python -m uvicorn main:app --port 8000 --reload
```

## 2. Li Experience Persistence (полностью реализовано)

### Проблема
Опыт Li-центров (`strength`, `accessCount`, `lastAccessed`) жил только в RAM и терялся при перезапуске.

### Решение

| Файл | Изменения |
|------|-----------|
| **core/src/centers/Li.ts** | `LearnOptions`, `getKnowledge()`, `learn()` с `restore` и возвратом `Knowledge` |
| **core/src/networks/BaseNetwork.ts** | `learn()` принимает `restore`, возвращает `Knowledge` |
| **core/src/persistence/KnowledgeStore.ts** | `last_accessed` в `KnowledgeRow`, flush/loadAll/snapshot |
| **core/src/MirrorAI.ts** | `learn()` передаёт реальные значения в enqueue; `init()` передаёт restore при загрузке |

### Поток данных
```
learn() → Li обновляет strength, accessCount, lastAccessed
       → KnowledgeStore.enqueue() с реальными значениями
       → Supabase li_knowledge (strength, access_count, last_accessed)

init() → loadAll() из Supabase
       → net.learn(..., { strength, accessCount, lastAccessed })
       → Li восстанавливает опыт
```

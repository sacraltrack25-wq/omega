## 1. Хранилище (Supabase Storage)

### 4 бакета

| Бакет | Доступ | Лимит | Назначение |
|---|---|---|---|
| `avatars` | публичный | 2 MB | Аватары профилей |
| `media` | приватный | 500 MB | Файлы пользователей для AI: фото, видео, аудио, PDF |
| `training-data` | только admins | 1 GB | Датасеты, curated Q&A, техники ответов |
| `knowledge-export` | только admins | 2 GB | Полные дампы Li знаний (backup/restore) |

### Новые SQL таблицы

**`query_attachments`** — прикреплённые файлы к запросам  
Когда пользователь отправляет фото/видео/аудио вместе с вопросом:
```
media/
  {user_id}/
    image/photo.jpg      → ImageNet
    video/clip.mp4       → VideoNet
    audio/song.mp3       → AudioNet
```
Поле `processed` меняется на `true` после того как AI Engine извлёк вектор.
Поле `vector_key` хранит ключ в `li_knowledge`.

**`curated_answers`** — лучшие ответы  
Администратор помечает образцовые ответы → они экспортируются в `training-data/` бакет как датасет для дообучения.

### Исправление миграции
Добавлены `DROP POLICY IF EXISTS` перед каждым `CREATE POLICY` — миграция теперь **idempotent** (можно запускать повторно без ошибок).

## 2. gX нейроны — Hebbian адаптация

**Файл:** `core/src/neurons/gX.ts`

### До (v1)
```
respond(bit: 0|1) → score: +1 | -1
```
Все нейроны равны. Один плохой нейрон влияет так же как хороший.

### После (v2)
```
respond(bit: 0|1) → score: +weight | -weight × 0.1
```

#### Ключевые изменения

**Вес нейрона `weight ∈ [0.1 .. 3.0]`**  
Инициализируется случайно `[0.8 .. 1.2]`. Меняется с каждым ответом:
- При резонансе: `weight × 1.008` (растёт)
- При промахе: `weight × 0.996` (падает)
- Streak-бонус: серия правильных активаций даёт дополнительный прирост

**Асимметричный score**  
```
primary resonance:  +weight        (сильный положительный сигнал)
mirror anti-res:    -weight × 0.1  (слабое подавление)
```
Нейрон не наказывает жёстко за ошибку — только мягко сомневается.  
Это моделирует реальную нейропластичность: "neurons that fire together, wire together".

**Внешний feedback**  
```typescript
neuron.strengthen(1.05)   // пользователь нажал 👍
neuron.weaken(0.95)       // пользователь нажал 👎
```
Feedback из `answer_feedback` таблицы буквально изменяет веса нейронов.

**Grow с наследованием веса**  
Новые нейроны при росте слоя стартуют со средним весом существующих — быстрее адаптируются.

## 3. Li центры — взвешенный резонанс

**Файл:** `core/src/centers/Li.ts`

### Fingerprint: float[] вместо binary[]

**До:** fingerprint = `[0, 1, 1, 0, 1, 0, ...]` (boolean активации)  
**После:** fingerprint = `[0.8, -0.1, 1.4, -0.05, 2.1, ...]` (weighted scores нейронов)

Непрерывный fingerprint несёт больше информации: не просто "нейрон сработал/не сработал", но и "насколько уверен был нейрон".

### gxResonanceSimilarity: cosine вместо bit-count

**До:**
```
matches = count(a[i] === b[i])
score   = matches / len          // просто совпадения 0/1
```

**После:**
```
cosine  = dot(a, b) / (|a| × |b|)   // направление fingerprint-векторов
score   = (cosine + 1) / 2           // нормировано [0..1]
```

Нейроны с большим весом вносят больший вклад в сходство.  
Смысл: два похожих запроса дадут fingerprints указывающие "в одну сторону".

### Экспоненциальное взвешивание слоёв

```
layer 0: importance = exp(0)   = 1.0   (меньше)
layer 1: importance = exp(0.5) = 1.65
layer 2: importance = exp(1.0) = 2.72  (больше, ближе к выходу)
```

Выходные слои важнее входных — как в реальных нейросетях.

## 4. Omega — softmax синтез

**Файл:** `core/src/omega/Omega.ts`

### До: линейное взвешивание
```
weight_i = confidence_i × (0.5 + agreement_i × 0.5)
output = Σ(weight_i × result_i) / Σ(weight_i)
```
Все Li кластеры могут влиять примерно одинаково.

### После: softmax
```
logit_i  = confidence_i × (0.5 + agreement_i × 0.5)
weight_i = exp(logit_i / T) / Σ exp(logit_j / T)
output   = Σ(weight_i × result_i)         // уже нормировано
```

**Temperature T = 1.0 (по умолчанию)**

| T | Поведение |
|---|---|
| T → 0 | winner takes all: самый уверенный Li доминирует |
| T = 1.0 | стандартный баланс |
| T → ∞ | демократия: все кластеры равноправны |

**Почему лучше:**  
Если один кластер имеет confidence = 0.9 а остальные 0.3, при softmax он получает ~80% веса. При линейном — только ~43%. Результат: более чёткие, уверенные ответы.

**Numerical stability:** `max(logits)` вычитается перед `exp()` — нет overflow при больших logit.

## 5. ImageNet — CLIP вместо гистограммы

**Файл:** `services/harvesters/harvesters/image_harvester.py`

| | До (v1) | После (v2) |
|---|---|---|
| Модель | Пиксельная гистограмма | CLIP ViT-B/32 |
| Размерность | 128-dim | 512-dim |
| Пространственность | Нет | 4 квадранта (fallback) |
| Семантика | Нет | Да: "cat" ≈ "кот" ≈ фото кошки |

**CLIP (Contrastive Language-Image Pre-Training)**  
Модель обучена сопоставлять изображения и текстовые описания.  
Один и тот же 512-dim вектор описывает AND визуальный контент AND смысл.

Установка:
```bash
pip install open-clip-torch
```

## 6. VideoNet — CLIP + Temporal Pooling

**Файл:** `services/harvesters/harvesters/video_harvester.py`

| | До (v1) | После (v2) |
|---|---|---|
| Метод | 8×8 thumbnail pixels | CLIP per-frame + temporal pooling |
| Кадров | до 32 | до 16 (качество > количество) |
| Размерность | 128-dim | 512-dim |
| Метаданные | frame_count | title, duration, description, uploader |

**Temporal Pooling:**
```python
weights = [1.0, 1.0, ..., 1.0]
weights[0]  = 1.5   # первый кадр (начало сцены/контекст)
weights[-1] = 1.5   # последний кадр (финал/вывод)
```

Первый и последний кадры часто содержат title card и conclusion — они важнее средних.

Итоговый вектор L2-нормализован.

## Архитектура потока данных

```
Пользователь пишет запрос
        │
        ▼
[FilterEngine.preQuery()]          ← блокировка запрещённого контента
        │
        ▼
[TextNet.encode()]                 ← n-gram → вектор запроса
        │
        ▼
[Li1 ↔ Li2 (mirror)]              ← оба кластера ищут в знаниях
   gX fingerprint cosine similarity  (float[], не bool[])
   weightedResonance (взвешенный по слоям)
        │
        ▼
[Omega.synthesize()]               ← softmax(confidence × agreement)
   T = 1.0 (temperature)
        │
        ▼
[FilterEngine.postQuery()]         ← quality_min, source_boost, response_inject
        │
        ▼
[KnowledgeStore.enqueue()]         ← сохранение в Supabase li_knowledge
        │
        ▼
Ответ пользователю
```

## Как запускать

### SQL (один раз в Supabase Dashboard)

Запустить в порядке в SQL Editor:

```
001_initial.sql           → profiles, auth trigger
002_ai_engine.sql         → networks, li_knowledge, omega_queries, harvester_jobs
003_subscriptions.sql     → plans, subscriptions, conversations
004_knowledge_filters.sql → filters, answer_feedback
005_storage_buckets.sql   → 4 бакета + query_attachments + curated_answers
```

Файл `005_storage_buckets.sql` теперь **idempotent** — безопасно запускать повторно.

### Dev-запуск

```powershell
# 1. AI Engine (порт 4000)
pnpm --filter @omega/core dev

# 2. Web (порт 3001)
pnpm --filter @omega/web dev

# 3. Harvesters (порт 8000, опционально)
cd services/harvesters
uvicorn main:app --port 8000 --reload
```

### Установка CLIP для видео/изображений

```bash
cd services/harvesters
pip install open-clip-torch
```

Модель `ViT-B-32` (~350 MB) скачается автоматически при первом запуске.

## Что делает OMEGA уникальным

| Функция | Transformer (GPT/Claude) | OMEGA Mirror AI |
|---|---|---|
| Знания | Заморожены при обучении | **Живые** — растут в runtime |
| Обучение | Gradient descent (месяцы) | **Hebbian** — каждый запрос |
| Ошибки | Галлюцинации без проверки | **Mirror check**: Li1 ↔ Li2 |
| Feedback | Нет прямого влияния | 👍👎 буквально меняют веса нейронов |
| Прозрачность | Чёрный ящик | Видно: resonance%, какие источники |
| Цена | Огромные GPU постоянно | Малые GPU, масштаб по запросу |
